{[%hh_hdr_en:Common tasks :: Using external tools]}

PhoA is capable of attaching external programs to use for viewing, printing or processing of the photo album pictures.

This is implemented with so-called tools, which you create and configure.

There is a variety of tool kinds.

The major tool kind is represented by so-called default actions. Additionally, there's a possibility of using an arbitrary program as a custom tool. {[%hh_h2:Default actions]}

It is a sort of "simplified" tool kinds.

As is known, Windows introduces a concept of file type, which unambiguously linked to the file's extension. For instance, files with extension .doc are of type "Microsoft Word document", while .html files are Internet documents.

Most registered file types (at least document types) in Windows have an associated application, which opens the files of that type when you run them from the Explorer. This is how works the default action for that type of file. So, .doc files are open in Microsoft Word, while .html files are handled by the default browser.

Just such a behaviour is implemented by the Default action tool kind.

Besides there's a number of other standard actions, namely:

  • open;
  • edit;
  • print.

Each of those actions has a corresponding tool kind.

Any of the actions listed above may be customized by you using standard Windows tools. You do it by selecting the Tools | Folder options.. Explorer menu item, on the File types page.

Many graphic applications (such as graphic editors or viewers; but not PhoA) change file associations to open with themselves. Some of them confirm before making such changes.

Therefore if you are happy with your current file associations you will likely use just those standard tool kinds.

There's one more subtle advantage of standard actions over custom ones (described below). The point is that standard actions in Windows often use so-called DDE (Dynamic Data Exchange) commands, which allow opening several pictures by the same application at once, while custom tools often spawn a new application instance for each picture. The latter case is less convenient. {[%hh_h2:Заданное действие]}

Этот вид инструментов позволяет задать любую программу для открытия (или обработки) заданного типа файлов. Чаще всего это будет графический редактор или средство просмотра, но может быть и более специфический инструмент, например, программа пакетной обработки изображений. {[%hh_h2:Пункт-разделитель]}

Это самый незамысловатый вид инструмента и нужен он для создания разделителей в меню.

Так, с видами инструментов мы разобрались. Теперь есть смысл кратко описать {[%hh_h2:Маски инструментов]}

Чтобы задать применимость инструментов к разным видам изображений, в PhoA каждому инструменту сопоставлена так называемая маска, которая и определяет, может ли инструмент быть применён к файлам определённого типа. Правила составления масок во многом похожи на соответствующие правила Windows, и используются такие маски во многих местах программы. Подробное описание правил составления масок файлов приведено здесь.

Инструмент становится доступным только когда все выделенные изображения подходят под его маску (точнее, маски, так как их может быть несколько). {[%hh_h2:Использование инструмента]}

Чтобы инструмент был полезен, нужно, чтобы как минимум имелась возможность его вызвать. В PhoA инструменты могут появляться в различных меню. В каких именно — задаётся в его свойствах. {[%hh_h2:Настройки пользовательского действия]}

Как было сказано выше, для данного вида инструментов необходимо указать программу, а также некоторые дополнительные параметры. Подробное описание свойств инструментов приведено здесь.

Указанной программе передаётся в качестве параметра командной строки полный путь к файлу изображения. Большинство программ обработки изображений воспринимают таким образом файлы для открытия или обработки. {[%hh_ftr_en]}
